PROJECT 01 · 캡스톤 디자인 · 5인 프로젝트

원자의 세계 속으로

몰입형 VR 기술을 활용한 디지털 화학 교육 콘텐츠

C# (Scripting) Unity 2022.3.42f1 (LTS) Meta XR All-in-One SDK Visual Studio 2022 Meta Quest 2 / 3

주제 및 목표

교과서 속 평면적인 원자를 가상 공간(VR)에 구현하여 학습자의 직관적 이해를 돕습니다. 보이지 않는 미시 세계를 직접 손으로 다루는 체험형 시나리오를 통해 화학에 대한 흥미를 유발하는 것이 목표입니다.

  • 몰입형 환경 실감형 3D 학습 환경 구축
  • 상호작용 극대화 컨트롤러가 없는 Hand Tracking 최적화
  • 체험형 시나리오 직접 원자를 만지고 결합하는 경험 제공

프로젝트 개요

  • 개발 기간 2025.01.08 ~ 2025.05.28
  • 팀 구성 5인 팀 (협업)
  • 담당 업무 개발 총괄, HandTracking 개발
  • GitHub SEJINLEE01/Capstone

핵심 시스템 및 알고리즘 구현

1. 복합 조건 논리 연산을 활용한 제스처 인식 시스템

의도치 않은 시스템 오작동을 막기 위해 다중 조건을 감지하도록 설계했습니다.

  • ShapeRecognizer를 통해 다섯 손가락이 모두 굽혀진(Closed) '주먹' 형태를 수동적으로 정의합니다.
  • TransformRecognizer를 이용해 손바닥이 얼굴을 향하는지(Towards Face), 손가락이 위를 향하는지(Fingers Up)의 방향성을 판별합니다.
  • 위 조건들을 Active State Group의 AND 논리 연산으로 묶어, 두 조건이 완벽히 만족할 때만 Selector Unity Event Wrapper를 통해 주기율표 UI 호출(Spawn) 이벤트를 트리거합니다.

2. 물리 기반 직관적 상호작용 (Grab & Poke)

기존 레이저 포인터 방식(Ray Interaction)이 주는 물리적 이질감을 없애기 위해 사용자의 손을 가상 세계에 직접 투영했습니다.

  • Grab: 원자를 직접 손으로 쥐고 옮길 수 있는 파지 인터랙션을 활성화하여 높은 몰입감을 제공합니다.
  • Poke: Meta SDK의 Poke Interaction을 도입하여 가상 버튼에 물리적 깊이감을 부여하고, 실제 스위치를 찌르는 듯한 본능적인 조작법을 구현했습니다.

3. 분자 결합 판정 알고리즘

원자 간의 결합 과정을 실시간으로 추적하고 동적으로 시각화합니다.

  • OnTriggerEnter/Exit를 통해 결합 보드(제작판) 위의 원자 개수를 실시간으로 카운팅합니다.
  • 필요한 Target Name과 충돌 객체의 Tag를 1:1로 검사하여 엉뚱한 결합을 방지합니다.
  • 결합 판정 시, 현재 위치(FromObj)에서 중심점(ToObj)까지의 방향 벡터(Direction Vector)를 산출 후 정규화(normalized)하여, Update 루프에서 원자들이 자연스럽게 모여드는 결합 애니메이션을 수학적으로 구현했습니다.

Trouble Shooting: 다중 객체 인식 시 UI 중복 출력 문제

🔥 문제 상황 (Problem)

초기 설계에서는 if 조건문과 count 변수를 활용하여 한 번에 하나의 정보창만 띄우도록 했습니다. 하지만 실제 환경에서 여러 원자가 센서(결합대)에 동시에 진입할 경우, UI 캔버스가 겹쳐서 중복 출력되는 버그가 발생했습니다.

🔍 원인 분석 (Cause)

프레임 단위의 연산 속도 때문이었습니다. 두 원자가 거의 동시에 진입하면, count 변수가 업데이트되어 if문 조건이 막히기 전에 동일한 프레임 내에서 두 원자의 OnCollisionEnter 이벤트가 순차적으로 모두 실행되면서 방어 로직이 뚫렸습니다.

💡 해결 방안 (Solution)

단순 제어문으로는 찰나의 동시 입력을 막을 수 없다고 판단하여 큐(Queue) 자료 구조 기반의 대기열 시스템을 구축했습니다.

  • 신호가 오면 즉시 UI를 만들지 않고 모두 Queue<string>에 Enqueue합니다.
  • 시스템은 큐의 맨 앞 값(Peek)만 확인하여 순차적으로 화면에 하나씩 출력합니다.
  • Dictionary<string, GameObject>를 활용하여 원자가 영역을 벗어나면 대기열과 딕셔너리에서 즉각 제거하여 런타임 성능 최적화 및 중복 생성을 원천 차단했습니다.
✨ 회고 (Reflection)

명령어의 논리가 완벽해 보여도, 시스템 프레임 주기 내의 신호 중첩이 로직의 결함을 만들 수 있음을 배웠습니다. 복잡한 인터랙션 환경에서는 단순 조건문보다 자료 구조를 활용한 명확한 데이터 흐름 제어가 훨씬 효율적이고 안정적임을 깨달았습니다.
